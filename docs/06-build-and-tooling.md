### Targets

| Target | Type | Purpose |
|---|---|---|
| `sympp` | `SHARED` library (also `STATIC` via option) | The library |
| `sympp_tests` | executable | Catch2 test suite, links `sympp` + oracle helpers |
| `sympp_oracle` | INTERFACE | Header + helpers for talking to Python oracle |
| `sympp_bench` | executable | Google Benchmark perf suite |

## Dependencies

### Required (runtime)

| Dependency | Why | Source preference |
|---|---|---|
| **GMP** (`libgmp`, `libgmpxx`) | Arbitrary-precision integers and rationals | System / vcpkg / Conan |
| **MPFR** (`libmpfr`) | Arbitrary-precision floats | System / vcpkg / Conan |
| **fmtlib** *(if compiler lacks `<format>`)* | String formatting | FetchContent fallback |

### Required (test/dev only)

| Dependency | Why |
|---|---|
| **Python 3.11+** with `sympy` | Oracle harness |
| **Catch2 v3** | Test framework |
| **Google Benchmark** | Performance suite |
| **pybind11** | Optional Python bindings (Phase 16+) |

### Optional

| Dependency | Why |
|---|---|
| **LLVM** (‚â• 17, ORC JIT) | `lambdify` JIT backend (Phase 13) |
| **TBB** | Concurrent expression cache (alternative to sharded `std::unordered_map`) |
| **gnuplot** / **matplotlib-cpp** | Plotting bridge consumers (Phase 14) |

### Dependency strategy

1. **Try `find_package` first.** Honors system installs and package managers (vcpkg, Conan, Homebrew, apt).
2. **`FetchContent` fallback** for header-only deps (Catch2, fmtlib, Google Benchmark) so a fresh clone builds without setup.
3. **Never vendor GMP/MPFR/LLVM** ‚Äî let the user provide. Documented install commands per platform.

### Install commands per platform

```bash
# macOS (Homebrew)
brew install cmake gmp mpfr llvm python@3.12
pip3 install sympy

# Ubuntu
sudo apt install cmake libgmp-dev libmpfr-dev llvm-17-dev python3-pip
pip3 install sympy

# Windows (vcpkg)
vcpkg install gmp mpfr llvm
pip install sympy
```

## Public headers

### Export rules

- Public headers live under `include/sympp/<module>/`.
- Always include via angle brackets: `#include <sympp/core/symbol.hpp>`.
- One umbrella `<sympp/sympp.hpp>` for users who don't care about granularity.
- All public symbols in `namespace sympp`. MATLAB-named facade in `sympp::matlab`.
- `SYMPP_EXPORT` macro for shared-library symbol visibility (Windows DLL + GCC `-fvisibility=hidden`).

### Forward-declaration header

`<sympp/fwd.hpp>` ‚Äî declares `Basic`, `Expr`, etc. for users to consume in their own headers without full include cost.

## CMake `find_package` for consumers

A consumer should be able to:

```cmake
find_package(SymPP 1.0 REQUIRED)
target_link_libraries(my_app PRIVATE SymPP::sympp)
```

Provided by:
- `cmake/SymPPConfig.cmake.in` template ‚Üí `SymPPConfig.cmake` at install.
- `SymPPTargets.cmake` (auto-generated by `install(EXPORT ...)`).
- pkg-config `sympp.pc` file.

## CI

GitHub Actions, `.github/workflows/ci.yml`. One workflow with the matrix from [05-validation-strategy](05-validation-strategy.md), plus a separate `nightly.yml` that:
- Pulls `sympy@latest` and reruns oracle suite.
- Runs benchmarks on a self-hosted runner with stable hardware.
- Produces a perf-trend report.

### Per-PR checks
- Build (full matrix).
- Unit + oracle tests.
- clang-format diff.
- clang-tidy diff (only on changed files; full sweep on main).
- Coverage report (Codecov).

### Branch protection
- `main`: requires green CI, 1 review, no force-push.
- Tagged releases (`v*.*.*`) trigger a separate workflow that builds prebuilt binaries, generates docs, publishes vcpkg/Conan PRs.

## Static analysis

- **clang-format** ‚Äî config in `.clang-format` (LLVM-derived; column 100; trailing return types where clarifying).
- **clang-tidy** ‚Äî config in `.clang-tidy`. Active checks: `bugprone-*`, `cert-*`, `cppcoreguidelines-*` (with `-cppcoreguidelines-pro-bounds-pointer-arithmetic` and a few others off where they false-positive on hot paths), `modernize-*`, `performance-*`, `readability-*`. Disable `readability-identifier-length`.
- **include-what-you-use** ‚Äî run on PR (advisory; not a gate).
- **cppcheck** ‚Äî opt-in on demand.

## Sanitizers

- `SYMPP_ENABLE_ASAN=ON` ‚Äî AddressSanitizer.
- `SYMPP_ENABLE_UBSAN=ON` ‚Äî UB Sanitizer.
- `SYMPP_ENABLE_TSAN=ON` ‚Äî ThreadSanitizer (run cache concurrency tests under this).
- Enabled in CI via `Debug+ASan+UBSan` matrix slot.

## Versioning

- **Semantic versioning.** v1.0 = first stable API. Pre-1.0 may break headers freely.
- Version macros in `<sympp/version.hpp>`: `SYMPP_VERSION_MAJOR/MINOR/PATCH/STRING`.
- ABI: shared library `SOVERSION = MAJOR`, `VERSION = MAJOR.MINOR.PATCH`.
